# Weblog 上线前安全核验清单（2026-03-29）

## 使用方式

- 按顺序执行，每项执行后勾选。
- 任何一项失败：禁止上线，先修复再回归。
- 建议在预发布环境执行一次完整清单，生产发布前再执行一次关键项（P0）。

---

## A. 基础配置核验（P0）

- [ ] **生产环境 profile 正确**：`spring.profiles.active=prod`。
- [ ] **开发绕过开关关闭**：`blog.security.dev-bypass-enabled=false`。
- [ ] **JWT 与验证码密钥已替换为强随机值（>=32）**。
- [ ] **反向代理传递真实 IP 配置正确**（`X-Forwarded-For` / `X-Real-IP`）。
- [ ] **前端 CSP 阶段符合计划**：`NUXT_CSP_STAGE`（`report-only` / `dual` / `enforce`）。

通过标准：配置值与发布单一致，且应用启动日志无安全自检异常。

---

## B. 核心接口防护回归（P0）

## B1 `check-email` 验证码与限流

- [ ] 无 `X-Captcha-Token` 必须拒绝。
- [ ] 伪造 `X-Captcha-Token` 必须拒绝。
- [ ] 同 IP + 同域名连续高频请求触发限流。

示例命令：

```bash
curl -s -X POST "http://localhost:9091/api/portal/auth/check-email" \
  -H "Origin: http://localhost:3000" \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com"}'

curl -s -X POST "http://localhost:9091/api/portal/auth/check-email" \
  -H "Origin: http://localhost:3000" \
  -H "X-Captcha-Token: invalid-token" \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com"}'
```

通过标准：分别返回缺少令牌/令牌无效/限流错误码，不返回 `200`。

## B2 阅读控制与防绕过

- [ ] `access/read/{不存在postId}` 必须拒绝。
- [ ] 同会话改 `User-Agent`，`readCount` 不可重置。
- [ ] 新会话（新 cookie）计数重置，符合设备识别策略。

示例命令：

```bash
curl -s -X POST "http://localhost:9091/api/portal/access/read/999999999" \
  -H "Origin: http://localhost:3000"
```

通过标准：不存在文章返回 `40402`；同会话换 UA 后 `allowed` 仍为 `false`（读满后）。

## B3 搜索输入防护

- [ ] 超长关键词（>80）被拒绝。
- [ ] 控制字符关键词被拒绝。

通过标准：返回参数错误，不进入正常搜索结果。

## C. 前端渲染安全核验（P1）

- [ ] 文章详情 `MdPreview` 已启用 sanitize。
- [ ] 专题详情 `MdPreview` 已启用 sanitize。
- [ ] 所有 `v-html` 使用统一净化工具，不允许裸渲染用户输入。

代码点核验：

- `weblog-user/pages/post/[slug].vue`
- `weblog-user/pages/topic/[id].vue`
- `weblog-user/utils/security/xss.ts`

通过标准：恶意 Markdown/HTML 样本不执行脚本，仅文本化显示或被过滤。

---

## D. 安全响应头与浏览器策略（P1）

- [ ] 响应头存在 `X-Frame-Options: SAMEORIGIN`。
- [ ] 响应头存在 `X-Content-Type-Options: nosniff`。
- [ ] 响应头存在 `Referrer-Policy` 与 `Permissions-Policy`。
- [ ] CSP 灰度策略符合阶段（`dual` 阶段应同时下发 enforce + report-only）。

示例命令：

```bash
curl -s -D - -o /dev/null "http://localhost:3000"
```

通过标准：响应头与发布计划一致，无缺失。
