# 安全漏洞报告

如果你发现安全漏洞，请**不要**通过公开 Issue 报告。

请发送邮件至仓库维护者的私人邮箱，描述：

1. 漏洞类型和影响范围
2. 复现步骤
3. 受影响的版本
4. 建议的修复方案（如有）

我们会在收到报告后 48 小时内确认，并在修复后公开致谢。

## 安全最佳实践（部署者）

- 生产环境必须修改所有默认密码和密钥
- `.env.prod` 文件权限设为 `600`
- 不要将 `.env`、`.env.prod` 文件提交到版本控制
- 定期更新依赖，关注安全公告
- 生产环境关闭 Knife4j API 文档
- 确保 Nginx 配置了安全响应头（HSTS、CSP、X-Frame-Options 等）
